# USER.md — 피그말리온 유저 모델

> Hermes Agent의 "dialectical user modeling" 원칙 적용.
> 이 파일은 피그말리온이 누구인지 + 우리의 관계가 어떻게 진화했는지를 추적한다.
> 작업이 완료될 때마다 업데이트한다.

---

## 피그말리온 프로파일

```
역할: 룸메이커 채널 운영자, 방탈출 인터랙티브 콘텐츠 제작자
플랫폼: Cowork 환경 (Claude AI)
작업 방식: AI 에이전트들에게 방향을 주고 채워달라는 스타일
```

---

## 확인된 취향

| 카테고리 | 선호 | 비선호 |
|---|---|---|
| 스토리 장르 | 스릴러 (확인됨) | 미확인 |
| 반전 방식 | 복선 기반 반전, 파트너 배신 구조 | 꿈에서 깨기, 예측 가능한 반전 |
| 난이도 | ⭐⭐⭐⭐ (어려움) | 미확인 |
| 챕터 구성 | 12장 + 프롤로그 + 다분기 엔딩 (3개) | 단선형 엔딩 |
| 퀴즈/게임 비율 | 퀴즈 7 : 미니게임 5 | 미확인 |
| 전달 방식 | 에이전트 메모 형식으로 퀴즈/게임 위치 표시 | 미확인 |

---

## 작업 방식 패턴

```
- 큰 방향을 먼저 정하고 디테일은 에이전트에게 위임
- "오케이"로 다음 단계 진행, "수정해줘 + [내용]"으로 수정
- 한 번 방향이 정해지면 빠르게 진행 선호
- AI 에이전트가 주도적으로 제안하는 방식 선호
- 에이전트 구조 세팅에 관심 (Hermes Agent 참고 지시)
```

## 현재 관계 단계

```
단계: 초반 (작업 2회 이하)

현재 상태:
- 스릴러 장르 1편 완성
- 에이전트 구조 세팅 중
- 아직 장르 다양성, 세부 피드백 데이터 적음

다음 단계로 가려면:
- 피드백 3회 이상 누적
- 장르 2개 이상 작업
- 성공/실패 패턴 구분 가능해야 함
```

---

## 업데이트 규칙

- 피그말리온이 "수정해줘 + [내용]" 할 때마다 → 취향 데이터 업데이트
- 피그말리온이 "좋아" / "소름" / "이런 거 더" 할 때마다 → 성공 패턴 추가
- 피그말리온이 새 장르나 방식을 요청할 때마다 → 프로파일 확장
- 분기 선택 패턴이 쌓이면 → 선호 엔딩 유형 추론
