옴니버스 방탈출 시리즈 "색깔없는세상" (빨강·노랑·파랑·초록·하양, 각 30문제) + 파이널 "색깔있는세상"(인스타그램 형식 40문제, 미제작). [[roommaker-platform]]에 등재됨.

구조: `룸메이커_플랫폼/games/`에 `colorless-core.js`(공용 엔진 v2) + `colorless-style.css`(공용 스타일) + 색깔별 HTML(데이터 인라인, core/style 참조). 기존 플랫폼 게임들과 달리 단일파일이 아닌 공유 엔진 구조. 기획 문서는 `색깔없는세상_기획/` 폴더 (00~07번 md).

핵심 시스템: ① PRISM 입장코드 — 각 에피 클리어 시 P·R·I·S·M 조각을 localStorage `cw_prism`에 자동 저장, 5개 완성 시 파이널 무료 입장(파이널 정답 RAINBOW) ② 채색 시스템 — 세상은 흑백, 진행도에 따라 캐릭터 아바타(SVG)와 UI 색이 점점 물듦 ③ 미션형 인터랙션 — 키패드 금고, 핫스팟 탐색, 밸런스/타이밍/기억 미니게임.

**Why:** 사용자(피그말리온)의 초기 비전 — "세상은 전부 흑백인데 그 색깔 캐릭터만 점점 물들어가는 표현" + "'빛이 먼저 왔다'처럼 퀴즈가 아니라 미션/장치 조작 위주" 피드백을 v2에서 반영함. **v4 핵심 원칙(가장 중요)**: 반전을 중간에 흘리지 않음 — 처음부터 끝까지 어린아이 시점의 해맑은 이야기로만 진행("빨강이는 사람들을 웃음 짓게 하는 일을 해!"), 복선은 무해한 디테일로만 심고, 채색 100% 순간 엔딩 몽타주(이미지)에서 일괄 리빌 후 다음 세계로. 중간 리빌 연출(일기 다시 읽기 진실 공개, 섬뜩한 톤) 전면 금지. 기획서: 색깔없는세상_기획/09_v4_스토리_재기획_해맑음봉인.md
**How to apply:** 이 시리즈 수정 시 core.js를 고치면 5개 게임 전체 반영. 새 에피소드/파이널도 같은 엔진 재사용. 콘텐츠 수위: 파랑 에피소드는 방법·수단 묘사 금지, 은유만 사용, 엔딩에 상담 안내(109) 유지. OneDrive 마운트는 동기화 지연이 있어 bash 조립보다 Write 도구로 직접 쓰는 것이 안전.
